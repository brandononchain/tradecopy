TradingView Alert Message Templates
✅ For Pine Script strategies — symbol, direction, and size are all pulled dynamically from the strategy. Works for any instrument automatically.
{"action":"{{strategy.order.action}}","symbol":"{{ticker}}","qty":"{{strategy.order.contracts}}","price":"{{close}}","sl":"{{plot_0}}","tp":"{{plot_1}}","comment":"{{strategy.order.comment}}"}
Field reference:
{{strategy.order.action}} — automatically buy or sell depending on which side the strategy fired
{{ticker}} — the chart symbol (e.g. US100, EURUSD, XAUUSD) — works for any instrument
{{strategy.order.contracts}} — size from your strategy's order quantity setting
{{plot_0}} / {{plot_1}} — optional SL/TP from plotted values; remove these lines if not using them
ℹ️ For manual alerts (price crossovers, indicator conditions, etc.) where you hardcode the direction. Create one alert per direction — TradingView doesn't auto-fill buy/sell for non-strategy alerts.
BUY Alert Message
{"action":"buy","symbol":"{{ticker}}","qty":"1","price":"{{close}}"}
SELL Alert Message
{"action":"sell","symbol":"{{ticker}}","qty":"1","price":"{{close}}"}
To change size: replace "qty":"1" with any number — e.g. "qty":"0.5" for half a lot, "qty":"2" for 2 lots.
Symbol is always dynamic — {{ticker}} picks up whatever chart the alert fires on, so one alert can cover multiple instruments if you apply it to multiple charts.
With SL/TP: add "sl":"1.0800","tp":"1.1000" as fixed prices.
ℹ️ Send a close action to close an open position. The broker adapter will close the matching open position for that symbol.
{"action":"close","symbol":"{{ticker}}","qty":"0"}
Note: Close signals require the broker adapter to support position lookup. Currently supported for DX Trade and Tradovate. For MT4/MT5 use the EA's built-in close logic.

ℹ️ Works with any broker running DX Trade CFD (FTMO, MyForexFunds, and others). The Host is the domain you log into — e.g. dxtrade.ftmo.com or trade.yourbroker.com. The Vendor field is usually the broker subdomain prefix (e.g. ftmo) — leave blank if unsure.
Instrument IDs (optional — only needed for unlisted symbols)

DX Trade requires a numeric instrument ID alongside the symbol name. Common ones are pre-loaded. To find a missing ID: open DevTools on your broker's DX Trade platform → Network tab → place a manual trade → inspect the POST to /api/orders/single → note the instrumentId field.
